## 데이터 전처리 (Data Preprocessing)

**데이터를 분석 및 모델 학습을 위해 원본 데이터를 정제하고 변환하는 과정**

| 용어 | 의미 |
| --- | --- |
| 데이터 | 문자, 숫자, 소리, 그림, 영상, 단어 등의 형태로 된 의미 단위 |
| 전처리 | 다루기 쉬운 형태로 변환시키는 과정 |
| 데이터 전처리 | 분석 및 모델 학습을 위해 원본 데이터를 정제하고 변환하는 과정 |

### 사용 이유

**데이터의 품질을 높여 분석 및 머신러닝 모델의 성능을 향상시키기 위함**

| 이유 | 설명 |
| --- | --- |
| 모델 성능 향상 | 전처리를 통해 모델의 예측력과 성능을 높일 수 있음 |
| 신뢰성 있는 예측 | 결측값 처리와 이상치 제거를 통해 데이터의 신뢰성 확보 |
| 데이터 최적화 | 스케일링, 정규화, 변환 등을 통해 모델이 잘 처리할 수 있는 형태로 최적화 |

### 주요 전처리 작업

| 작업 | 설명 | 발생 원인 |
| --- | --- | --- |
| 결측값 처리 | 누락된 데이터를 식별하고 적절한 값으로 대체 | 데이터 수집 오류, 응답자 누락, 시간적 이유 등 |
| 이상치 제거 | 데이터 내 비정상적인 값을 식별하고 제거 | 데이터 입력 오류, 계측기기 오류, 특정 이벤트 등 |
| 데이터 정규화 | 데이터를 일정 범위(예: 0~1)로 변환 | 데이터 일관성, 알고리즘 성능 향상 |
| 데이터 변환 | 로그 변환, 원-핫 인코딩 등으로 변환 | 정규 분포화, 비선형 관계 처리 |

> **원-핫 인코딩**: 범주형 데이터를 이진 벡터로 변환하는 기법
> 예) `['사과', '바나나', '딸기']` → 사과: `[1, 0, 0]`, 바나나: `[0, 1, 0]`, 딸기: `[0, 0, 1]`
> 모델은 "사과", "바나나" 같은 문자열을 직접 처리할 수 없으므로 수치형으로 변환 필요
> 단순히 0, 1, 2로 매핑하면 모델이 크기 순서가 있다고 잘못 해석 → 원-핫 인코딩으로 해결

> **스케일링**: 다양한 수치 데이터를 동일한 범위나 척도로 변환하는 과정
> - **표준화(Standardization)**: 평균 = 0, 표준편차 = 1로 변환 — `(x - 평균) / 표준편차`
> - **정규화(Normalization)**: 특정 범위(0~1)로 변환 — `(x - min) / (max - min)`

> **이미지 전처리**: 일반적으로 이미지 데이터를 벡터(픽셀 값 배열)로 변환
> - 대부분의 머신러닝 알고리즘은 벡터 형태의 입력을 필요로 하기 때문

### 전처리 6단계 코드 예시

#### 1. 데이터 수집

```python
import pandas as pd

data = {
    '직원': ['John', 'Anna', 'Peter', 'Linda', 'James'],
    '연봉': [50000, 60000, 55000, None, 65000],
    '근속연수': [5, 7, None, 8, 9]
}
df = pd.DataFrame(data)
print(df)
```

```
     직원       연봉  근속연수
0   John  50000.0   5.0
1   Anna  60000.0   7.0
2  Peter  55000.0   NaN
3  Linda      NaN   8.0
4  James  65000.0   9.0
```

#### 2. 결측값 처리

결측값(NaN)을 중앙값(median)으로 대체

```python
df.loc[:, '연봉']   = df['연봉'].fillna(df['연봉'].median())
df.loc[:, '근속연수'] = df['근속연수'].fillna(df['근속연수'].median())
```

| 메서드 | 설명 |
| --- | --- |
| `fillna(값)` | NaN을 지정한 값으로 대체 |
| `median()` | 데이터를 오름차순 정렬했을 때 중앙에 위치한 값 — 이상값이 많을 때 평균보다 안전 |
| `df.loc[:, '열']` | 원본 데이터프레임의 해당 열을 직접 수정 |

#### 3. 이상치 제거

```python
df = df[df['연봉'] > 0]  # 연봉이 0 이하인 비정상 데이터 제거
```

#### 4. 데이터 정규화

```python
from sklearn.preprocessing import MinMaxScaler

scaler = MinMaxScaler()
df[['연봉', '근속연수']] = scaler.fit_transform(df[['연봉', '근속연수']])
```

| 메서드 | 설명 |
| --- | --- |
| `MinMaxScaler()` | 최소값과 최대값 기반으로 각 값을 0~1 사이로 변환 |
| `fit_transform()` | 데이터를 학습(fit)하고 변환(transform)을 동시에 수행 |

```
    직원      연봉   근속연수
0   John  0.000000  0.000
1   Anna  0.666667  0.500
2  Peter  0.333333  0.625
3  Linda  0.500000  0.750
4  James  1.000000  1.000
```

#### 5. 데이터 변환 (원-핫 인코딩)

```python
df = pd.get_dummies(df, columns=['직원'])
```

`pd.get_dummies()`: 지정 열을 원-핫 인코딩하여 각 범주를 이진 열로 변환, 원본 열은 제거됨

```
        연봉   근속연수  직원_Anna  직원_James  직원_John  직원_Linda  직원_Peter
0  0.000000  0.000    False     False     True     False     False
1  0.666667  0.500     True     False    False     False     False
...
```

#### 6. 데이터 분할

```python
from sklearn.model_selection import train_test_split

train, test = train_test_split(df, test_size=0.2, random_state=42)
```

| 파라미터 | 설명 |
| --- | --- |
| `test_size=0.2` | 전체의 20%를 테스트 데이터로 분할 |
| `random_state=42` | 난수 시드 고정 — 동일 값 설정 시 항상 같은 결과 재현 가능 |

## 데이터 증강 (Data Augmentation)

**머신러닝을 위해 기존 데이터를 변형하여 새로운 데이터를 생성하는 기법**

원본 데이터를 단순 복사하는 것이 아니라, 회전·크기 조절·색상 변화·노이즈 추가 등의 방법으로 다양한 형태의 데이터를 생성

### 사용 이유

| 이유 | 설명 |
| --- | --- |
| 데이터 다양성 증가 | 다양한 형태의 데이터를 제공하여 모델이 다양한 상황에 잘 일반화되도록 함 |
| 일반화 성능 향상 | 과적합을 완화하고 새로운 데이터에도 잘 대응하는 모델 생성 |
| 데이터 부족 문제 해결 | 실제 데이터가 부족한 상황에서 증강 기법으로 더 많은 데이터 확보 가능 |
| 모델 견고성 향상 | 노이즈나 변형된 데이터에도 잘 대응하는 모델 생성 |

### 증강 기법

| 기법 | 설명 |
| --- | --- |
| 이미지 회전 (Rotation) | 이미지를 특정 각도로 회전 |
| 이미지 뒤집기 (Flipping) | 좌우 또는 상하 반전 |
| 이미지 이동 (Translation) | 픽셀을 x, y 방향으로 이동 |
| 크기 조절 (Rescaling) | 확대 또는 축소 |
| 색상 변형 (Color Jittering) | 밝기, 대비, 채도 조절 |
| 노이즈 추가 (Adding Noise) | 랜덤 픽셀 변화를 추가하여 노이즈에 강한 모델 학습 |
| 역번역 (Back-Translation) | [NLP] 다른 언어로 번역 후 재번역 |
| 속도 변화 (Speed Perturbation) | [음성] 오디오 파일의 속도 변경 |

### 한계

| 한계 | 설명 |
| --- | --- |
| 원본과 완전히 동일하지 않음 | 잘못된 변형 적용 시 실제 데이터 특성을 반영하지 못하고 학습이 왜곡될 수 있음 |
| 특정 변형에 의존할 위험 | 특정 변형된 데이터에는 강하지만 완전히 새로운 데이터에 취약할 수 있음 |
| 모든 도메인에 적합하지 않음 | 금융 데이터, 정형 데이터 등에서는 증강 효과가 제한적 |

### 코드 예시

```python
from PIL import Image, ImageOps, ImageEnhance
import numpy as np

def create_random_image(size=(100, 100)):
    # np.random.randint(0, 256, size+(3,)): 100x100x3 배열 생성 (RGB)
    # dtype=np.uint8: 픽셀 값(0~255) 표현에 적합한 타입
    return Image.fromarray(np.random.randint(0, 256, size + (3,), dtype=np.uint8))

image = create_random_image()

# 1. 이미지 회전 — 45도 반시계 방향
rotated = image.rotate(45)

# 2. 이미지 뒤집기
flipped_h = ImageOps.mirror(image)  # 좌우 반전
flipped_v = ImageOps.flip(image)    # 상하 반전

# 3. 이미지 이동
# np.roll: 배열을 지정된 축을 따라 이동 (axis=0: 행/y축, axis=1: 열/x축)
array = np.array(image)
translated = Image.fromarray(np.roll(array, shift=(30, 50), axis=(0, 1)))

# 4. 이미지 크기 조정
width, height = image.size
resized = image.resize((int(width * 1.5), int(height * 1.5)))

# 5. 밝기 조절
enhancer = ImageEnhance.Brightness(image)
brightened = enhancer.enhance(1.5)  # 1.0이 원본, 1.5는 50% 밝게
```

> **axis=0이 행(y축)인 이유**: `arr[i][j]`에서 i가 변하면 위아래(행)로 이동하므로
> axis=0 = 행 방향(세로/y축), axis=1 = 열 방향(가로/x축)
> → `shift=(y, x)` 순서로 작성

## 데이터셋 분할 (Dataset Split)

**주어진 데이터를 학습(training) · 검증(validation) · 테스트(test)로 분할하여 모델의 성능을 평가하고 일반화하기 위한 과정**

| 구분 | 설명 |
| --- | --- |
| 학습(training) 데이터 | 모델이 패턴을 학습하는 데 사용되는 데이터 |
| 검증(validation) 데이터 | 하이퍼파라미터 튜닝 및 모델 최적화 과정에서 사용 — 선택 사항 |
| 테스트(test) 데이터 | 학습이 완료된 모델의 최종 성능을 평가하는 데이터 — 한 번도 본 적 없는 데이터여야 함 |

### 사용 이유

| 이유 | 설명 |
| --- | --- |
| 과적합 방지 | 학습 데이터에만 치우치지 않고 새로운 데이터에도 잘 일반화되도록 함 |
| 성능 평가 | 모델의 성능을 객관적으로 측정 |
| 일반화 확인 | 모델이 실제 데이터를 처리할 때 잘 일반화되는지 검증 |
| 하이퍼파라미터 튜닝 | 검증 데이터로 모델의 설정값을 최적화 |

> **일반화**: 모델이 학습에 사용되지 않은 새 데이터에도 유사한 성능을 내는 능력
> 특정 훈련 데이터에만 맞춰진 패턴을 학습하지 않도록 하는 것이 핵심

### 분할 비율

| 분할 비율 | 학습 | 검증 | 테스트 | 설명 |
| --- | --- | --- | --- | --- |
| 80:20 | 80% | - | 20% | 검증 없이 테스트만 사용 |
| 70:30 | 70% | - | 30% | 테스트 비율을 더 크게 설정 |
| 60:20:20 | 60% | 20% | 20% | 일반적인 3분할 방식 |
| 70:15:15 | 70% | 15% | 15% | 검증 포함 3분할 방식 |

> 8:2, 7:3은 절대 규칙이 아닌 **통계적 타당성과 실무 효율성이 굳어진 관행**
> 소규모 데이터는 90%에서 교차검증 + 최종 평가용 10% 보존 방식이 권장됨

### 방법 1 — 직접 분할

인덱스 슬라이싱으로 순서대로 분할 (데이터 순서가 중요한 경우 사용)

```python
import pandas as pd

data = {
    'feature1': range(1, 101),
    'feature2': range(101, 201),
    'label': [1 if x % 2 == 0 else 0 for x in range(1, 101)]
}
df = pd.DataFrame(data)

train_size      = int(0.6 * len(df))  # 60%
validation_size = int(0.2 * len(df))  # 20%

train_data      = df[:train_size]                                  # 0~59행
validation_data = df[train_size:train_size + validation_size]      # 60~79행
test_data       = df[train_size + validation_size:]                # 80~99행

print(f"학습: {len(train_data)}, 검증: {len(validation_data)}, 테스트: {len(test_data)}")
# 학습: 60, 검증: 20, 테스트: 20
```

### 방법 2 — 랜덤 분할 (train_test_split)

데이터를 무작위로 섞어서 분할 — 일반적으로 가장 많이 사용

```python
from sklearn.model_selection import train_test_split

# 학습 80% / 테스트 20%
train_data, test_data = train_test_split(df, test_size=0.2, random_state=42)

# 학습 60% / 검증 20% / 테스트 20%
train_data, temp_data       = train_test_split(df, test_size=0.4, random_state=42)
validation_data, test_data  = train_test_split(temp_data, test_size=0.5, random_state=42)
```

| 파라미터 | 설명 |
| --- | --- |
| `test_size` | 분할 비율 (0.2 = 20%) |
| `random_state` | 난수 시드 — 동일 값 사용 시 항상 같은 분할 결과 보장 (재현성 확보) |

> **random_state를 설정하는 이유**: 시드를 지정하지 않으면 실행할 때마다 다른 분할이 발생
> → 결과 검증·디버깅·모델 비교 시 공정한 비교가 불가능해짐

### 시나리오별 분할 전략

| 시나리오 | 전략 |
| --- | --- |
| 데이터셋이 하나만 있는 경우 | `train_test_split`으로 학습/테스트 분할, 필요시 학습에서 검증 추가 분할 |
| 학습·테스트가 이미 분리된 경우 | 학습 데이터에서만 검증 데이터를 추가 분할, 테스트 데이터는 최종 평가용으로 보존 |
| 데이터가 충분히 분리된 경우 | 추가 분할 없이 그대로 사용 |

> 테스트 데이터는 **학습 과정에서 절대 사용하지 않아야 함** — 모델이 한 번도 보지 않은 데이터로 최종 성능을 평가해야 신뢰성 확보 가능
