# 8-2. 벡터 유사도 계산

## 개요

벡터 유사도 계산은 임베딩된 텍스트 간의 관련성을 수치적으로 비교하는 방법입니다.

RAG에서는 사용자 질문을 임베딩한 벡터와 문서 청크의 벡터를 비교하여 가장 관련성이 높은 문서를 찾습니다.

대표적인 방법으로 Dot Product와 Cosine Similarity가 있습니다.

---

# Dot Product (내적)

## 정의

Dot Product는 두 벡터의 같은 위치 원소끼리 곱한 뒤 모두 더하여 하나의 스칼라 값을 만드는 연산입니다.

간단하면서도 계산이 매우 빠르기 때문에 벡터 검색과 Transformer의 Attention 연산에서 널리 사용됩니다.

## 핵심 개념

- 같은 위치 원소끼리 곱한 후 합산
- 결과는 하나의 숫자(스칼라)
- 벡터의 방향과 크기 모두 영향을 받음
- 계산 비용이 낮음

### 계산 예시

A = [1, 2, 3]

B = [4, 5, 6]

Dot Product:

1×4 + 2×5 + 3×6 = 32

## 사용 이유

- 계산이 매우 빠름
- GPU 병렬 처리에 적합
- 대규모 벡터 검색에 활용 가능
- Transformer의 Scaled Dot-Product Attention 핵심 연산

## 사용 방법

1. 두 벡터 준비
2. 같은 위치 원소끼리 곱하기
3. 모든 결과를 더하기

### 코드 예시

```python
import numpy as np

a = np.array([1, 2, 3])
b = np.array([4, 5, 6])

result = np.dot(a, b)

print(result)
```

출력:

```python
32
```

## 한계

Dot Product는 벡터 크기의 영향을 받습니다.

방향이 같더라도 벡터 크기가 크면 더 높은 점수를 받을 수 있습니다.

따라서 순수한 의미 유사도를 비교할 때는 Cosine Similarity가 더 많이 사용됩니다.

---

# Cosine Similarity (코사인 유사도)

## 정의

Cosine Similarity는 두 벡터가 이루는 각도의 코사인 값을 이용하여 유사도를 계산하는 방법입니다.

벡터의 크기를 제거하고 방향만 비교하기 때문에 텍스트 임베딩 비교에 가장 널리 사용됩니다.

## 핵심 개념

- 방향의 유사도만 비교
- 벡터 크기의 영향 제거
- 결과 범위는 -1 ~ 1
- RAG 검색에서 가장 많이 사용

### 수식

Cosine Similarity =

(A · B) / (||A|| × ||B||)

여기서

- A·B : Dot Product
- ||A|| : 벡터 A의 크기(L2 Norm)
- ||B|| : 벡터 B의 크기(L2 Norm)

## 결과 해석

| 값 | 의미 |
|------|------|
| 1에 가까움 | 매우 유사 |
| 0에 가까움 | 관련성 낮음 |
| -1에 가까움 | 반대 방향 |

## 사용 이유

- 벡터 크기에 영향받지 않음
- 텍스트 길이가 달라도 비교 가능
- 결과 해석이 쉬움
- 벡터 데이터베이스에서 널리 사용

## 사용 방법

1. Dot Product 계산
2. 각 벡터 크기 계산
3. 크기의 곱으로 나누기

### 코드 예시

```python
import numpy as np

def cosine_similarity(a, b):
    return np.dot(a, b) / (
        np.linalg.norm(a) *
        np.linalg.norm(b)
    )

a = np.array([1, 2, 3])
b = np.array([4, 5, 6])

print(
    cosine_similarity(a, b)
)
```

## RAG에서 활용

```text
사용자 질문
↓
임베딩
↓
질문 벡터 생성
↓
문서 벡터와 Cosine Similarity 계산
↓
상위 K개 문서 선택
↓
LLM 입력
```

# Dot Product vs Cosine Similarity

| 항목 | Dot Product | Cosine Similarity |
|--------|--------|--------|
| 비교 기준 | 방향 + 크기 | 방향 |
| 결과 범위 | 무한대 | -1 ~ 1 |
| 크기 영향 | 있음 | 없음 |
| 계산 속도 | 더 빠름 | 약간 느림 |
| RAG 활용 | 일부 사용 | 가장 많이 사용 |

## 선택 기준

### Dot Product

- 정규화된 벡터 사용
- 빠른 검색이 중요
- Attention 계산

### Cosine Similarity

- 일반적인 RAG
- 의미 기반 검색
- 문서 유사도 비교

---

# 실무 팁

## 정규화된 임베딩

일부 임베딩 모델은 출력 벡터를 자동으로 정규화합니다.

이 경우 Dot Product와 Cosine Similarity가 거의 동일한 결과를 반환할 수 있습니다.

## 임계값 사용

실무에서는 Cosine Similarity 점수를 기준으로 관련성이 낮은 문서를 제거합니다.

예시:

- 0.8 이상 : 매우 유사
- 0.5 ~ 0.8 : 관련 있음
- 0.3 ~ 0.5 : 약한 관련성
- 0.3 이하 : 관련성 낮음

다만 절대 기준은 아니며 데이터와 모델에 따라 달라집니다.

## RAG 검색 품질 향상

Cosine Similarity만 사용해도 충분한 경우가 많지만, 검색 품질을 높이기 위해 다음 기법을 함께 사용합니다.

- Hybrid Search
- Query Expansion
- Reranking
- HyDE

# 핵심 정리

- Dot Product는 두 벡터의 대응 원소를 곱한 후 모두 더하는 연산이다.
- Dot Product는 벡터 크기의 영향을 받는다.
- Cosine Similarity는 Dot Product를 벡터 크기로 나누어 정규화한 값이다.
- Cosine Similarity는 방향만 비교한다.
- RAG에서는 Cosine Similarity가 가장 널리 사용되는 유사도 계산 방식이다.
