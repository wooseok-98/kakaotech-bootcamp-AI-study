# 8-3. Chunking

## Chunking 개요

Chunking은 긴 문서를 검색과 임베딩에 적합한 크기의 텍스트 조각(Chunk)으로 분할하는 과정입니다.

### 사용하는 이유
- 임베딩 모델 입력 길이 제한 해결
- 검색 정확도 향상
- LLM에 필요한 문맥만 전달 가능
- 출처 추적 용이

### 주요 파라미터
- `chunk_size` : 청크 크기
- `chunk_overlap` : 인접 청크 간 중복 구간

일반적으로 overlap은 chunk_size의 10~20% 정도를 사용합니다.

---

## Fixed-size Chunking

미리 정한 글자 수 또는 토큰 수 기준으로 문서를 분할하는 방식입니다.

### 장점
- 구현이 가장 단순
- 빠른 프로토타이핑 가능
- 문서 구조에 관계없이 적용 가능

### 단점
- 문장 중간에서 잘릴 수 있음
- 문맥 손실 가능성 존재

### 적합한 경우
- RAG 초기 구축
- 구조가 없는 일반 텍스트

---

## Regex & Delimiter Chunking

줄바꿈, 마침표, 제목 등의 구분자를 기준으로 문서를 분할하는 방식입니다.

### 주요 구분자
- 빈 줄 (`\n\n`)
- 줄바꿈 (`\n`)
- 마침표 (`.`)
- Markdown 제목 (`#`, `##`)
- 정규표현식 패턴

### 장점
- 자연스러운 문맥 유지
- 구현이 비교적 단순
- 추가 임베딩 비용 없음

### 단점
- 청크 크기가 일정하지 않음
- 문서 구조가 없으면 효과 감소

### 적합한 경우
- Markdown 문서
- HTML 문서
- 법률 문서
- 기술 문서

---

## Semantic Chunking

문장 간 임베딩 유사도를 이용하여 의미가 변하는 지점에서 문서를 분할하는 방식입니다.

### 동작 과정
1. 문장 분리
2. 문장 임베딩 생성
3. 인접 문장 유사도 계산
4. 유사도가 낮은 지점 탐색
5. 의미 단위 청크 생성

### 장점
- 주제 단위 분할 가능
- 검색 정확도 향상
- 주제 변화 자동 감지

### 단점
- 임베딩 비용 발생
- Threshold 튜닝 필요
- 처리 시간이 증가

### 적합한 경우
- 회의록
- 뉴스 기사
- 다양한 주제가 섞인 문서

---

## Parent-Child Chunking

검색용 작은 청크(Child)와 문맥 제공용 큰 청크(Parent)를 함께 관리하는 방식입니다.

### 구조
- Parent : 넓은 문맥 보관
- Child : 검색 수행

### 동작 방식
1. 문서를 Parent로 분할
2. Parent를 Child로 재분할
3. Child만 임베딩
4. 검색 수행
5. 검색된 Child의 Parent를 LLM에 전달

### 장점
- 높은 검색 정밀도
- 풍부한 문맥 제공
- 검색과 문맥의 균형 확보

### 단점
- 구현 복잡
- Parent-Child 관계 관리 필요

### 적합한 경우
- 매뉴얼
- 보고서
- 기술 문서
- 대규모 RAG 시스템

---

## 전략 비교

| 전략 | 검색 정확도 | 문맥 보존 | 비용 | 난이도 |
|--------|--------|--------|--------|--------|
| Fixed-size | 중간 | 낮음 | 매우 낮음 | 낮음 |
| Regex & Delimiter | 중간 | 높음 | 매우 낮음 | 낮음 |
| Semantic | 높음 | 높음 | 높음 | 높음 |
| Parent-Child | 높음 | 매우 높음 | 중간~높음 | 높음 |

---

## 실무 가이드

- 처음 구축할 때는 Fixed-size Chunking부터 시작
- 구조화 문서는 Regex & Delimiter Chunking 사용
- 검색 품질이 중요하면 Semantic Chunking 적용
- 대규모 서비스에서는 Parent-Child Chunking 고려

### 권장 시작값

| 용도 | Chunk Size | Overlap |
|--------|--------|--------|
| 일반 QA | 500~1000자 | 10~20% |
| 문서 요약 | 1000~2000자 | 10~20% |
| 코드 문서 | 함수/클래스 단위 | 필요 시 적용 |
