# FastAPI 백엔드 개발 가이드

---

## 백엔드 개발 흐름

백엔드 개발은 크게 이 순서로 진행

```
1. 어떤 기능이 필요한지 파악
2. 필요한 API 엔드포인트 설계 (URL, Method, 요청/응답 형태)
3. 프로젝트 구조 잡기
4. 코드 구현 (Router → Controller → Model 순서)
5. 테스트
```

---

## 단계별 사용 언어 및 기술

### 1단계: 기능 파악

특별한 언어나 기술 없음. 어떤 기능이 필요한지 글로 정리하는 단계

```
예) 커뮤니티 서비스
- 회원가입 / 로그인
- 게시글 목록, 상세 조회
- 게시글 작성, 수정, 삭제
- 내가 쓴 글 보기
```

---

### 2단계: API 설계

- **언어**: 없음 (설계 문서 작성)
- **사용 기술**: REST API 설계 규칙, HTTP 메서드, HTTP 상태 코드

REST API 설계 규칙에 따라 각 기능을 URL + HTTP 메서드로 표현

```
기능           HTTP 메서드  URL
게시글 목록    GET          /posts
게시글 작성    POST         /posts
게시글 수정    PUT          /posts/{id}
게시글 삭제    DELETE       /posts/{id}
```

각 API마다 어떤 데이터를 받고 반환할지도 정의

```
POST /posts
  요청: { "title": "제목", "content": "내용" }
  응답: { "data": { "id": 1, "title": "제목", "content": "내용" } }
```

### 4단계: 코드 구현

- **언어**: Python
- **사용 기술**: FastAPI, Pydantic

각 파일별로 사용하는 기술이 다름

#### dtos/ — 요청/응답 데이터 형태 정의

- **기술**: Pydantic (`BaseModel`)
- FastAPI에서 요청 데이터 구조와 타입을 정의
- 선언한 타입과 다른 값이 오면 자동으로 422 에러 반환

```python
from pydantic import BaseModel

class PostCreate(BaseModel):
    title: str      # 문자열만 허용
    content: str    # 문자열만 허용
```

#### models/ — 데이터 저장/조회

- **기술**: Python (순수 함수 + 리스트/딕셔너리)
- 지금은 메모리(리스트)에 저장 → 나중에 DB 연동 시 이 파일만 교체
- 판단 없이 저장/조회 기능만 담당

```python
_posts = []   # 서버 재시작하면 데이터 사라짐 (지금은 이렇게 사용)

def get_all():
    return _posts

def create(title, content):
    post = {"id": ..., "title": title, "content": content}
    _posts.append(post)
    return post
```

#### controllers/ — 비즈니스 로직

- **기술**: Python + FastAPI (`HTTPException`)
- "없는 게시글이면 404 반환" 같은 판단과 처리를 담당
- Model을 호출해서 데이터를 가져오고, 응답 형태로 포장

```python
from fastapi import HTTPException
from models import post_model

def get_post(post_id):
    post = post_model.get_by_id(post_id)
    if not post:
        raise HTTPException(status_code=404, detail="post_not_found")
    return {"data": post}
```

#### routers/ — URL 경로 정의

- **기술**: FastAPI (`APIRouter`, 데코레이터)
- "이 URL로 오면 이 Controller 함수 실행해"만 담당
- 로직은 작성하지 않음

```python
from fastapi import APIRouter
router = APIRouter()

@router.get("/{post_id}")       # GET /posts/{post_id} 요청을 받으면
def get_post(post_id: int):
    return post_controller.get_post(post_id)    # controller로 넘김
```

#### main.py — 앱 생성 및 라우터 등록

- **기술**: FastAPI (`FastAPI()`, `include_router`)
- 각 라우터를 prefix와 함께 등록
- `prefix="/posts"` → router 안의 `@router.get("")`이 `GET /posts`가 됨

```python
from fastapi import FastAPI
app = FastAPI()
app.include_router(post_router, prefix="/posts")
```

## REST API 설계 방법

### 1단계: 리소스 파악

서비스에서 다루는 데이터(리소스)가 무엇인지 먼저 정의

예) 커뮤니티 서비스라면 → **사용자(User)**, **게시글(Post)**

### 2단계: 엔드포인트 설계

각 리소스에 대해 필요한 동작을 HTTP 메서드 + URL로 표현

```
리소스: Post (게시글)

GET    /posts           → 목록 조회
GET    /posts/{id}      → 상세 조회
POST   /posts           → 생성
PUT    /posts/{id}      → 전체 수정
PATCH  /posts/{id}      → 부분 수정
DELETE /posts/{id}      → 삭제
```

### 3단계: 요청/응답 형태 정의

각 엔드포인트에서 어떤 데이터를 주고받는지 정의

```
POST /posts
  요청: { "title": "제목", "content": "내용" }
  응답: { "data": { "id": 1, "title": "제목", "content": "내용" } }

GET /posts/{id}
  요청: 없음 (URL에 id만)
  응답: { "data": { "id": 1, "title": "제목", "content": "내용" } }
```

### Pydantic - 요청 Body 정의

FastAPI에서 요청 데이터의 구조와 타입을 정의하는 방법  
선언한 타입과 다른 값이 오면 **자동으로 422 반환**

```python
# dtos/post_dto.py
from pydantic import BaseModel

class PostCreate(BaseModel):
    title: str
    content: str

class PostUpdate(BaseModel):
    title: str
    content: str
```

### Router - URL 경로 정의

```python
# routers/post_router.py
from fastapi import APIRouter
from controllers import post_controller
from dtos.post_dto import PostCreate, PostUpdate

router = APIRouter()

@router.get("")                        # GET /posts
def get_posts():
    return post_controller.get_posts()

@router.get("/{post_id}")              # GET /posts/1
def get_post(post_id: int):
    return post_controller.get_post(post_id)

@router.post("", status_code=201)      # POST /posts
def create_post(payload: PostCreate):
    return post_controller.create_post(payload)

@router.put("/{post_id}")             # PUT /posts/1
def update_post(post_id: int, payload: PostUpdate):
    return post_controller.update_post(post_id, payload)

@router.delete("/{post_id}", status_code=204)  # DELETE /posts/1
def delete_post(post_id: int):
    post_controller.delete_post(post_id)
```

### Controller - 비즈니스 로직

```python
# controllers/post_controller.py
from fastapi import HTTPException
from models import post_model
from dtos.post_dto import PostCreate, PostUpdate

def get_posts():
    return {"data": post_model.get_all()}

def get_post(post_id: int):
    post = post_model.get_by_id(post_id)
    if not post:
        raise HTTPException(status_code=404, detail="post_not_found")
    return {"data": post}

def create_post(payload: PostCreate):
    post = post_model.create(payload.title, payload.content)
    return {"data": post}

def update_post(post_id: int, payload: PostUpdate):
    post = post_model.get_by_id(post_id)
    if not post:
        raise HTTPException(status_code=404, detail="post_not_found")
    updated = post_model.update(post_id, payload.title, payload.content)
    return {"data": updated}

def delete_post(post_id: int):
    post = post_model.get_by_id(post_id)
    if not post:
        raise HTTPException(status_code=404, detail="post_not_found")
    post_model.delete(post_id)
```

### Model - 데이터 저장/조회

지금은 리스트로 저장 (나중에 DB로 교체)

```python
# models/post_model.py
_posts = []
_next_id = 1

def get_all():
    return _posts.copy()

def get_by_id(post_id: int):
    return next((p for p in _posts if p["id"] == post_id), None)

def create(title: str, content: str):
    global _next_id
    post = {"id": _next_id, "title": title, "content": content}
    _posts.append(post)
    _next_id += 1
    return post

def update(post_id: int, title: str, content: str):
    post = get_by_id(post_id)
    if post:
        post["title"] = title
        post["content"] = content
    return post

def delete(post_id: int):
    global _posts
    _posts = [p for p in _posts if p["id"] != post_id]
```

