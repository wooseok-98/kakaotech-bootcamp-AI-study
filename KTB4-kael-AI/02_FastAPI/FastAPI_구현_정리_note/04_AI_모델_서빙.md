## 1. AI 서빙이란

**AI 서빙 (AI Serving)** — AI 모델을 API 형태로 제공해서 다른 서비스가 호출할 수 있게 하는 것

```
브라우저 → FastAPI 서버 → Ollama (LLM) → 요약 결과 반환
```

우리 FastAPI 서버는 클라이언트(브라우저)의 요청을 받아서, Ollama에 다시 요청을 보내고, 결과를 돌려주는 **중간 다리** 역할을 한다.

백엔드 엔지니어의 역할을 데이터 물류에 비유하면:

```
FE → BE 서버 → AI 서버 → DB / LLM
         ↑ JSON 상하차 ↑
```

## 2. Ollama란

**Ollama** — 내 컴퓨터에서 LLM을 로컬로 실행할 수 있는 도구

| 항목 | 설명 |
|------|------|
| 설치 | `https://ollama.com` 에서 다운로드 |
| 모델 다운로드 | `ollama pull gemma3:1b` |
| 서버 주소 | `http://localhost:11434` |
| 인터넷 필요 여부 | 모델 다운로드 시에만 필요. 이후는 로컬에서 동작 |

### 모델 선택 기준

| 모델 | 크기 | 특징 |
|------|------|------|
| `gemma3:1b` | 815MB | 빠름. 간단한 요약에 적합 |
| `gemma3:4b` | ~3GB | 더 자연스러운 결과 |
| `llama3.2:3b` | ~2GB | 영어 성능 좋음 |

숫자가 클수록 성능은 좋지만 느리고 메모리를 많이 쓴다.

## 3. httpx란

**httpx** — 파이썬에서 HTTP 요청을 보내는 라이브러리

FastAPI 서버가 Ollama 서버에 요청을 보낼 때 사용한다. 브라우저가 `fetch()`로 서버에 요청하는 것처럼, 파이썬 코드도 다른 서버에 HTTP 요청을 보낼 수 있다.

```bash
pip install httpx
```

```python
import httpx

# GET 요청
res = httpx.get("http://localhost:11434/api/tags")

# POST 요청 (JSON body 포함)
res = httpx.post(
    "http://localhost:11434/api/chat",
    json={"model": "gemma3:1b", "messages": [...], "stream": False},
    timeout=60   # LLM은 응답이 느리므로 timeout을 길게 설정
)

data = res.json()   # 응답 JSON 파싱
```

## 4. Ollama API 호출하기

Ollama는 `http://localhost:11434` 에서 HTTP API를 제공한다.

### 채팅 API (`/api/chat`)

```python
import httpx

res = httpx.post(
    "http://localhost:11434/api/chat",
    json={
        "model": "gemma3:1b",
        "messages": [
            {"role": "user", "content": "안녕하세요, 자기소개 해줘."}
        ],
        "stream": False   # True면 응답을 조금씩 흘려줌 (스트리밍)
    },
    timeout=60
)

result = res.json()
print(result["message"]["content"])  # LLM의 답변
```

### stream=False vs stream=True

| 항목 | stream=False | stream=True |
|------|-------------|-------------|
| 응답 방식 | 완성된 전체 응답을 한 번에 | 글자가 생성되는 대로 조금씩 |
| 구현 난이도 | 단순 | 복잡 (이벤트 루프 필요) |
| 사용자 경험 | 기다렸다가 한 번에 | 타이핑하듯 즉시 표시 |

우리 구현은 `stream=False`를 사용한다.

## 5. 우리 코드에서 어떻게 쓰이는가

### 파일 구조

```
code_AI/
├── ai/
│   └── llm.py                 # Ollama 호출 담당
├── controllers/
│   └── summary_controller.py  # DB 조회 + llm.py 호출
└── routers/
    └── summary_router.py      # URL 연결
```

### ai/llm.py — Ollama 호출

```python
import httpx

OLLAMA_URL = "http://localhost:11434/api/chat"
MODEL = "gemma3:1b"

def ask(prompt: str) -> str:
    response = httpx.post(
        OLLAMA_URL,
        json={
            "model": MODEL,
            "messages": [{"role": "user", "content": prompt}],
            "stream": False,
        },
        timeout=60,
    )
    response.raise_for_status()        # 200이 아니면 에러 발생
    return response.json()["message"]["content"]
```

- `raise_for_status()` — 응답 코드가 4xx, 5xx면 예외를 발생시킨다. 에러를 조용히 무시하지 않기 위한 방어 코드.
- `timeout=60` — LLM 응답은 느리므로 충분한 시간을 줘야 한다.

### controllers/summary_controller.py — DB 조회 후 LLM 호출

```python
from models import post_model, comment_model
from ai import llm

def summarize_post(post_id: int):
    post = post_model.get_post_by_id(post_id)  # 1. DB에서 게시글 조회
    
    prompt = f"다음 게시글을 한국어로 3문장 이내로 요약해줘.\n\n제목: {post['title']}\n내용: {post['content']}"
    
    summary = llm.ask(prompt)  # 2. Ollama에 요약 요청
    
    return {"message": "게시글 요약 성공", "summary": summary}  # 3. 결과 반환
```

### routers/summary_router.py — URL 연결

```python
from fastapi import APIRouter
from controllers import summary_controller

router = APIRouter(prefix="/posts")

@router.get("/{post_id}/summary")
def summarize_post(post_id: int):
    return summary_controller.summarize_post(post_id)

@router.get("/{post_id}/comments/summary")
def summarize_comments(post_id: int):
    return summary_controller.summarize_comments(post_id)
```

### 전체 흐름

```
GET /posts/1/summary
      ↓
summary_router.py  (URL 연결)
      ↓
summary_controller.py  (post_model로 DB 조회 → prompt 생성 → llm.ask() 호출)
      ↓
ai/llm.py  (httpx로 Ollama에 POST 요청)
      ↓
Ollama  (gemma3:1b 모델이 요약 생성)
      ↓
{"message": "게시글 요약 성공", "summary": "..."}
```
