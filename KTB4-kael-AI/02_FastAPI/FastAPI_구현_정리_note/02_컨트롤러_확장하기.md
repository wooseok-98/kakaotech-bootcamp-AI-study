# 컨트롤러 확장하기 — Route-Controller-Model 패턴

---

## 디렉토리 구조

```
code_vanila/
├── main.py
├── routers/
│   └── user_router.py      # URL 연결만 담당
├── controllers/
│   └── user_controller.py  # 비즈니스 로직 담당
└── models/
    └── user_model.py       # 데이터 저장/조회 담당
```

---

## 요청 흐름

```
클라이언트 요청
      ↓
  main.py (라우터 등록)
      ↓
  routers/user_router.py (URL 매핑)
      ↓
  controllers/user_controller.py (로직 처리)
      ↓
  models/user_model.py (데이터 접근)
      ↓
  응답 반환
```

---

## 각 계층 코드

### main.py — 라우터 등록만 담당

```python
from fastapi import FastAPI
from routers import user_router

app = FastAPI()

app.include_router(user_router.router)
```

---

### routers/user_router.py — URL 연결만 담당

```python
from fastapi import APIRouter
from controllers import user_controller

router = APIRouter(prefix="/users")

@router.post("")            # POST /users   → 회원가입
def create_user(data: dict):
    return user_controller.create_user(data)

@router.post("/login")      # POST /users/login → 로그인
def login(data: dict):
    return user_controller.login(data)
```

- `prefix="/users"` — 이 파일의 모든 URL 앞에 `/users` 가 붙음
- 라우터는 로직을 처리하지 않고 컨트롤러에 넘기기만 함

---

### controllers/user_controller.py — 비즈니스 로직 담당

```python
from fastapi import HTTPException
from models import user_model

def create_user(data: dict):
    if user_model.get_users_by_email(data["email"]):
        raise HTTPException(status_code=409, detail="이미 사용 중인 이메일입니다.")

    if data["password"] != data["password_confirm"]:
        raise HTTPException(status_code=400, detail="비밀번호가 일치하지 않습니다.")

    new_user = {
        "id": len(user_model.get_all_users()) + 1,
        "email": data["email"],
        "password": data["password"],
        "nickname": data["nickname"],
        "profile_image": data.get("profile_image", None)
    }

    user_model.add_user(new_user)
    return {"message": "회원가입 성공", "user": new_user}

def login(data: dict):
    user = user_model.get_users_by_email(data["email"])
    if not user:
        raise HTTPException(status_code=404, detail="존재하지 않는 이메일입니다.")
    if user["password"] != data["password"]:
        raise HTTPException(status_code=401, detail="비밀번호가 틀렸습니다.")
    return {"message": "로그인 성공", "user": user}
```

- `HTTPException` — 에러 발생 시 상태 코드와 메시지를 담아 응답
- `data.get("profile_image", None)` — 없는 키를 꺼낼 땐 `.get()` 사용, 없으면 `None` 반환

| 상태 코드 | 의미 |
|-----------|------|
| 200 | 성공 |
| 400 | 잘못된 요청 (비밀번호 불일치) |
| 401 | 인증 실패 (비밀번호 틀림) |
| 404 | 찾을 수 없음 (없는 이메일) |
| 409 | 충돌 (이메일 중복) |

### models/user_model.py — 데이터 저장/조회 담당

```python
_users = []

def get_all_users():
    return _users

def get_users_by_id(user_id: int):
    for u in _users:
        if u["id"] == user_id:
            return u
    return None

def get_users_by_email(email: str):
    for u in _users:
        if u["email"] == email:
            return u
    return None

def add_user(user: dict):
    _users.append(user)
    return user
```

- `_users` — 언더스코어(`_`)로 시작하면 이 파일 내부에서만 쓰는 변수라는 관례
- for문으로 직접 순회해서 조건에 맞는 첫 번째 항목 반환, 없으면 `None` 반환

---

## 왜 이렇게 나누는가?

| 계층 | 담당 | 장점 |
|------|------|------|
| Router | URL 연결 | URL 구조만 보면 어떤 기능인지 파악 가능 |
| Controller | 로직 처리 | 유저 로직 수정 시 `user_controller.py`만 열면 됨 |
| Model | 데이터 관리 | DB로 교체할 때 모델만 수정하면 됨 |
