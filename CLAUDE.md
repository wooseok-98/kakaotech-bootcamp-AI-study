## 문서 작성 규칙

### 공통 문체

- **마침표(`.`) 제거** — 문장 끝에 마침표 사용 안 함
- **명사형 종결** — "~이다", "~한다" 대신 명사·체언으로 간결하게 마무리
- 핵심 개념은 **굵게** 강조
- 한국어 기준, 기술 용어는 영문 병기 (예: 전이 학습 (Transfer Learning))

### README 작성 방식

- 저장소·챕터 단위로 노트 목록을 테이블로 정리
- 테이블 컬럼: `파일` / `주제`
- 파일 링크는 마크다운 상대경로, 공백은 `%20` 인코딩
- 상단에 한 줄 소개 + 목차 테이블 배치

### 학습 노트 작성 방식

교재 원문을 받으면 핵심만 추출해 마크다운 노트로 정리

**포함**
- 개념 정의 (1~2줄)
- 핵심 특징·비교 테이블
- 코드 예시 (주석은 핵심 줄만)
- 중요한 부가 설명은 `>` 인용구

**제외**
- 5단 분석법 표 (명사 분리 설명)
- 저작권 문구, 이미지 첨부 태그
- Q&A 형식 부가 설명 (핵심만 본문 통합)

**파일 형식**

```
# {섹션}-{번호}. {주제명}
{1~2줄 설명}
> {추가 컨텍스트 (선택)}
---
## {섹션}
{표 또는 코드}
```

## 파일 네이밍 규칙

- 형식: `{섹션}-{번호}_{영문주제}.md` (예: `4-1_시계열데이터.md`)
- 관련 개념은 하나의 파일로 통합 (개념별 분리 지양)
- 챕터 요약 파일: `summary.md` 또는 `0-0_summary.md`
